## Assembly evaluation with KAT
**Reason for method choice:**
- KAT (K-mer Analysis Toolkit) compares k-mer spectra between raw reads and assemblies.
- It measures assembly completeness, detects missing regions, and can highlight misassemblies.
- The 'comp' mode checks which k-mers from the reads are present in the assembly and vice versa.
- A dedicated environment **kat_env** was created because KAT had strict dependencies (old Boost version) that conflicted with Python versions in 'assem' and 'eval' environments.
- This separation ensures we can run KAT without breaking other tools in the pipeline.

## BUSCO genome completeness assessment

**Rationale & setup:**
- BUSCO (Benchmarking Universal Single-Copy Orthologs) was used to assess the completeness of each assembly (Flye, Raven, Shasta) against the *bacteria_odb10* lineage dataset.
- We created a dedicated conda environment `busco_env` instead of using the KAT or assembly environments because BUSCO has many version-sensitive dependencies (Python, HMMER, BLAST, Prodigal, Augustus, SEPP) that can conflict with other tools.
- Initial attempts to install BUSCO from bioconda failed due to dependency resolution issues, so we downloaded BUSCO v5.8.0 directly from its GitLab repository and installed it manually.
- Missing dependencies (e.g., pandas, numpy, matplotlib, biopython, bbtools, requests) were installed incrementally as BUSCO reported them missing during runs.
- Installing `bbtools` was necessary because BUSCO uses it for pre-processing FASTA/FASTQ input.
- Installing `requests` resolved a Python import error when BUSCO attempted to fetch lineage datasets.
- The bacteria_odb10 dataset was chosen because our assemblies are bacterial in origin, making the completeness metrics biologically relevant.

**Environment creation & installation:**

conda create -y -n busco_env python=3.10 conda activate busco_env wget https://gitlab.com/ezlab/busco/-/archive/5.8.0/busco-5.8.0.tar.gz tar -xzf busco-5.8.0.tar.gz cd busco-5.8.0 conda install -y pandas numpy matplotlib biopython hmmer blast augustus prodigal sepp openmpi ucx conda install -y -c bioconda bbmap pip install requests pip install .

16S rRNA Extraction, BLASTN Identification, and Phylogenetic Analysis

Objective: To extract representative 16S rRNA sequences from each assembler output (Flye, Raven, Shasta), identify their closest bacterial species via BLASTN, and construct a phylogenetic tree for comparative analysis.

1. Environment Setup We created a dedicated environment to avoid package conflicts and ensure reproducibility:

conda create -y -n ssu_env -c bioconda barrnap bedtools seqkit blast mafft trimal iqtree entrez-direct
conda activate ssu_env
  • barrnap: Predicts rRNA genes (5S, 16S, 23S) using HMM models.
  • bedtools: Extracts sequences from assemblies using genomic coordinates.
  • seqkit: Sorts, filters, and manipulates FASTA/FASTQ files.
  • blast: Compares sequences against NCBI databases.
  • mafft: Performs multiple sequence alignment.
  • trimal: Removes poorly aligned regions to improve phylogenetic accuracy.
  • iqtree: Infers maximum likelihood phylogenetic trees.
  • entrez-direct: Fetches sequences directly from NCBI.

2. 16S rRNA Detection with barrnap We used barrnap to scan assemblies for bacterial 16S rRNA genes and output GFF files:

mkdir -p ~/flot_project/eval/16s
barrnap --kingdom bac ~/flot_project/assemblies/flye/assembly.fasta > ~/flot_project/eval/16s/flye.gff
barrnap --kingdom bac ~/flot_project/assemblies/raven/raven.fasta > ~/flot_project/eval/16s/raven.gff
barrnap --kingdom bac ~/flot_project/assemblies/shasta/Assembly.fasta > ~/flot_project/eval/16s/shasta.gff

We then filtered the GFF files to keep only 16S rRNA hits:

awk '$3=="rRNA" && /product=16S/' ~/flot_project/eval/16s/flye.gff > ~/flot_project/eval/16s/flye_16S.gff
awk '$3=="rRNA" && /product=16S/' ~/flot_project/eval/16s/raven.gff > ~/flot_project/eval/16s/raven_16S.gff
awk '$3=="rRNA" && /product=16S/' ~/flot_project/eval/16s/shasta.gff > ~/flot_project/eval/16s/shasta_16S.gff

3. Extracting 16S Sequences We used bedtools to extract nucleotide sequences for the predicted 16S rRNAs:

bedtools getfasta -fi ~/flot_project/assemblies/flye/assembly.fasta -bed ~/flot_project/eval/16s/flye_16S.gff -s > ~/flot_project/eval/16s/flye_16S.fasta
bedtools getfasta -fi ~/flot_project/assemblies/raven/raven.fasta -bed ~/flot_project/eval/16s/raven_16S.gff -s > ~/flot_project/eval/16s/raven_16S.fasta
bedtools getfasta -fi ~/flot_project/assemblies/shasta/Assembly.fasta -bed ~/flot_project/eval/16s/shasta_16S.gff -s > ~/flot_project/eval/16s/shasta_16S.fasta

We sorted sequences by length and kept only the longest per assembler:

for ASM in flye raven shasta
do
  seqkit sort -l -r ~/flot_project/eval/16s/${ASM}_16S.fasta | seqkit head -n 1 > ~/flot_project/eval/16s/${ASM}_16S_rep.fasta
done

4. BLASTN Species Identification We ran BLASTN for each representative 16S sequence against the NCBI nt database:

for ASM in flye raven shasta
do
  blastn -query ~/flot_project/eval/16s/${ASM}_16S_rep.fasta -db nt -remote \
  -entrez_query "Bacteria[ORGN] AND 16S[Title]" \
  -max_target_seqs 10 -evalue 1e-50 \
  -outfmt "6 qseqid sacc pident length qlen qcovs bitscore stitle" \
  > ~/flot_project/eval/16s/${ASM}_blastn.tsv
done

Explanation of key parameters:

  • -entrez_query: Restricts search to bacterial 16S sequences.
  • -max_target_seqs 10: Limits results to top 10 hits.
  • -evalue 1e-50: Ensures matches are highly significant.
  • -outfmt 6: Outputs tabular format with specific columns. Inspect Top Hit for Each Assembly Command:
for ASM in flye raven shasta; do
  head -n 1 ~/flot_project/eval/16s/${ASM}_blastn.tsv
done

Output:

Flye:   Acinetobacter baumannii (100.000% identity, 1529/1533 bp, strain 16842)
Raven:  Klebsiella pneumoniae (99.610% identity, 1537/1537 bp, strain NUBRI-K)
Shasta: Acinetobacter baumannii (99.804% identity, 1530/1534 bp, strain 16842)

** Building the Phylogenetic Tree** We retrieved reference sequences for the top 15 Flye BLASTN hits:

cut -f2 ~/flot_project/eval/16s/flye_blastn.tsv | head -n 15 | paste -sd, - | \
xargs -I{} sh -c 'efetch -db nucleotide -id {} -format fasta' > ~/flot_project/eval/16s/ref_seqs.fasta

We combined assembler sequences with reference sequences:

cat ~/flot_project/eval/16s/*_16S_rep.fasta ~/flot_project/eval/16s/ref_seqs.fasta > ~/flot_project/eval/16s/all_16S.fasta

We aligned sequences with MAFFT and trimmed with trimAl:

mafft --auto --thread $(nproc) ~/flot_project/eval/16s/all_16S.fasta > ~/flot_project/eval/16s/all_16S.aln.fasta
trimal -automated1 -in ~/flot_project/eval/16s/all_16S.aln.fasta -out ~/flot_project/eval/16s/all_16S.trim.fasta

We built the tree with IQ-TREE:

** Interpretation** Our comparative genome assembly and evaluation pipeline revealed distinct strengths and limitations among Flye, Raven, and Shasta, with downstream 16S rRNA analysis providing a crucial biological perspective. Flye generated the most contiguous genome (N50 ≈ 3.77 Mb, 2 contigs), near-complete k-mer recovery (99.85% completeness by KAT), and high BUSCO completeness (98.4%, no duplications), supported by a clean circular assembly graph. Raven matched Flye’s N50 but produced 28 contigs and exhibited an unusually high BUSCO duplication rate (44.4%), suggesting unresolved repeat collapse or redundancy despite excellent k-mer completeness (99.90%). Shasta, in contrast, produced the most fragmented assembly (N50 ≈ 437 kb, BUSCO 78.2%) with lower k-mer completeness (88.41%), yet intriguingly shared the same top 16S BLASTN hit as Flye (Acinetobacter baumannii, ~100% identity). Raven’s divergence (Klebsiella pneumoniae) despite high completeness underscores that single-gene taxonomic inference can be impacted by assembly artifacts or contamination, and that whole-genome quality does not always align with locus-specific phylogenetic placement.

🔍 Key Insight — Shasta–Flye Convergence Despite Divergent Quality Shasta’s markedly lower completeness but identical 16S rRNA match to Flye demonstrates that highly conserved loci can remain intact even in fragmented assemblies, provided the raw reads sufficiently cover those regions. This convergence, contrasted with Raven’s phylogenetic divergence, highlights the importance of integrating both genome-wide metrics (QUAST, KAT, BUSCO, Bandage) and locus-specific analyses (BLASTN, phylogeny) for robust, biologically meaningful taxonomic conclusions.

Genome Annotation and Metabolic Capacity Analysis (Prokka + BlastKOALA)

1. Environment Setup

We created a dedicated Conda environment to ensure Prokka ran with all necessary dependencies without conflicting with other tools in our workflow: conda create -n annotation_env -c bioconda -c conda-forge prokka conda activate annotation_env This isolated environment guaranteed reproducibility and minimized dependency issues.

2. Running Prokka

Prokka is a rapid prokaryotic genome annotation tool. It predicts coding sequences (CDS), rRNAs, tRNAs, and assigns functional annotations by comparing against curated databases. We annotated the assemblies from all three assemblers:

Flye

prokka ~/flot_project/assemblies/flye/assembly.fasta
--outdir ~/flot_project/eval/annotation/flye_prokka
--prefix flye

Raven

prokka ~/flot_project/assemblies/raven/assembly.fasta
--outdir ~/flot_project/eval/annotation/raven_prokka
--prefix raven

Shasta

prokka ~/flot_project/assemblies/shasta/assembly.fasta
--outdir ~/flot_project/eval/annotation/shasta_prokka
--prefix shasta

3. Key Output: The .faa Files

From Prokka, the most important output for this step was the .faa file:

  • This file contains all predicted protein sequences in FASTA format.
  • Proteins are the functional workhorses of the cell, and their sequences allow us to link genome data to metabolic pathways.
  • KEGG's BlastKOALA uses these sequences to assign KEGG Orthology (KO) numbers and map them to metabolic pathways.

4. Preparing Files for BlastKOALA

We copied the .faa files from our Linux environment to our local machine to upload them to BlastKOALA: cp ~/flot_project/eval/annotation/flye_prokka/flye.faa /mnt/c/Users/Louis/Downloads/ cp ~/flot_project/eval/annotation/raven_prokka/raven.faa /mnt/c/Users/Louis/Downloads/ cp ~/flot_project/eval/annotation/shasta_prokka/shasta.faa /mnt/c/Users/Louis/Downloads/ The /mnt/c/... path bridges the WSL/Linux environment to the Windows filesystem, placing the files into our Downloads folder for easy web upload.

5. Running BlastKOALA

We uploaded each .faa file to KEGG's BlastKOALA (http://www.kegg.jp/blastkoala/) using the 'Prokaryotes' database (refprok.pep). BlastKOALA matches protein sequences to KO terms and groups them into functional categories such as metabolism, genetic information processing, and environmental processing.

6. Results and Observations

Flye (Acinetobacter baumannii)

  • Proteins predicted: 3,425 | Annotated: 1,966 (57.4%)
  • Major categories: Protein families (signaling, cellular), genetic information processing, carbohydrate and amino acid metabolism, environmental information processing.
  • Interpretation: Rich functional diversity consistent with A. baumannii's adaptability and persistence in diverse environments.

Raven (Klebsiella pneumoniae)

  • Proteins predicted: 6,137 | Annotated: 3,552 (57.9%)
  • Functional distribution similar in shape to Flye's, but reflecting K. pneumoniae-specific traits, especially carbohydrate metabolism and membrane transport.
  • Interpretation: The divergence from Flye/Shasta matches earlier phylogeny results and suggests either contamination or assembler-specific bias.

Shasta (Acinetobacter baumannii)

  • Proteins predicted: 2,984 | Annotated: 1,617 (54.2%)
  • Functional profile strongly resembles Flye's, despite Shasta's much lower genome completeness.
  • Interpretation: Even fragmented assemblies can capture essential conserved loci and core metabolic pathways if coverage over these regions is adequate.

7. Integrated Project Interpretation

This BlastKOALA metabolic analysis forms the final, complementary layer of our project, linking structural genome data to biologicaion. From assembly to metabolic annotation, our pipeline demonstrated:

  • Flye consistently provided the most complete and contiguous genome, yielding the richest functional annotation for A. baumannii.
  • Shasta, although fragmented and incomplete, still converged with Flye at both the 16S phylogenetic and metabolic profile levels, proving the resilience of conserved gene regions.
  • Raven, despite excellent completeness scores from KAT and BUSCO, diverged in both phylogeny and metabolic signature, underlining that completeness alone does not ensure biological accuracy.

By integrating assembly metrics (QUAST, KAT, BUSCO), assembly structure (Bandage), taxonomic inference (BLASTN + phylogeny), and functional capacity (Prokka + BlastKOALA), we achieved a holistic view of each assembler's performance — technically and biologically. This final step confirms that genome assembly evaluation should always extend beyond structural metrics to include functional and metabolic perspectives for robust, real-world conclusions.
